# 每日时间块消消乐（dayplan-timeline）

一个**单文件**、**纯前端**、**零依赖**的每日时间块打卡 Web 应用。打开 `index.html` 即可使用。

把一天拆成若干个时间块，挨个点击完成；每完成一块会获得 1 枚金币；当日全部完成时显示庆祝页。原生支持**工作日 / 周末 / 出差**三种日程模板，可翻看历史打卡，状态全部保存在 `localStorage`。

## 快速开始

直接双击 `index.html` 即可在浏览器中运行。

如需用本地静态服务（可选）：

```bash
# 任选一种
npx serve .
python3 -m http.server 8080
```

> 首次打开时会在 `localStorage` 中自动创建当天计划；清空浏览器存储即可完全重置。

## 核心特性一览

- 📅 **四套日程模板**：工作日 / 周六 / 周日（按当前星期自动选择）/ 出差（顶栏勾选）
- ✅ **时间块打卡**：点击方块完成；动画上移 + 缩小（`popOut`），并出现 `+1 🪙` 浮动金币
- 🪙 **金币系统**：每完成一块 +1 枚；全局累计（顶栏右侧）；重置会扣回
- 🎉 **庆祝页**：当日全部完成后展示，可一键重置当日
- 🗓 **历史记录**：列出「至少完成过 1 个时间块」的日期，点击可重新查看
- 📱 **移动端优先**：窄屏（< 640px）Tab 文案自动缩短为「今日·工作日 / 历史」，进度文案缩短为 `1/3 · 6/2 · 工作日`
- 🛡 **无障碍**：`role="tab"`、`aria-selected`、`aria-label`、`aria-live`、键盘可达
- 🔄 **旧数据兼容**：旧版 `time` 字段、缺失字段、模板变更都会自动迁移，**保留同名块的打勾状态**
- 💾 **本地存储**：纯 `localStorage`，关闭浏览器数据仍在

## 界面布局

```
┌──────────────────────────────────────────┐
│  [今日打卡(周六)]  ☐ 出差  [历史记录]   │  ← Tab 栏（窄屏自动缩短）
├──────────────────────────────────────────┤
│   ‹   完成 0 / 0 · 6月2日 · 工作日   🪙0 │  ← 状态栏（无底色，融入背景）
├──────────────────────────────────────────┤
│  06:40          ●       游泳       30′   │
│  08:30          ●       聚焦工作    3h   │  ← 时间轴（左侧时间 · 中间节点 · 右侧卡片）
│  11:50          ●       午饭       30′   │
│  ...                                      │
└──────────────────────────────────────────┘
```

### 顶栏（Tab + 状态栏）

- **Tab 栏**（3 列 grid）：
  - 左侧：`今日打卡（工作日）` / `今日打卡（周六）` / `今日打卡（周日）` / `今日·工作日` / `今日·周六` / `今日·周日`（随当前查看日期动态变化）
  - 中间：`☐ 出差` 复选框（窄屏时变窄；查看历史时禁用）
  - 右侧：`历史记录` / `历史`
- **状态栏**（3 列 flex，无底色 / 边框 / 阴影，与背景融为一体）：
  - 左侧：日期切换 `‹` `›`（超过今天会隐藏后一天按钮）
  - 中间：进度文案 `完成 N / M · 6月2日 · 周六`（窄屏 `N/M · 6/2 · 周六`）
  - 右侧：累计金币 `🪙 N`

### 时间块卡片

每行 3 列：

| 列 | 内容 |
|---|---|
| 左侧 | 开始时间（11px 等宽字体，灰色） |
| 中间 | 时间轴上的圆形节点；未完成空心 + 浅灰；已完成实心 + 浅绿 |
| 右侧 | 卡片：未完成白底 / 已完成绿底 + 删除线 + `✓` 标记；副标题右侧是紧凑时长（`30′`、`3h`、`3h30′`） |

**时段块**（如「白天」「傍晚」）：
- 不显示时长
- 开始时间用普通字体（不再是等宽）

### 浮动金币动画

点击时间块后，从卡片中心弹出 `+1 🪙`，向上飘动 50px 并淡出（0.9s）。

### 庆祝页（全部完成时）

```
🎉
今日全部完成！
[重置当日]   ← 查看历史时为「重置当日」
```

### 历史记录页

- 顶部 `← 返回今日` 按钮
- 列表：每条 = 日期 + 工作流标签（`工作日` / `周六` / `周日` / `出差`）+ `已完成/总数` 绿底徽章
- 点击某条 → 跳回今日页并切换到该日期
- 暂无记录时显示 `暂无打卡记录 / 完成今日打卡后会自动保存`

## 行为约定

- **工作流选择**：
  - 周一至周五 → 工作日
  - 周六 → 周六模板（`|` / `/` 分隔符前半段）
  - 周日 → 周日模板（`|` / `/` 分隔符后半段）
  - 顶栏勾选「出差」→ 强制使用出差表（无论星期）
  - 取消勾选 → 回到按星期的默认表
  - 切换出差时若已有打卡进度，会弹窗确认；确认后清空当日并扣回已得金币
- **打卡权限**：
  - 只有「今天」可点击打卡
  - 查看历史日期时方块为只读（`.readonly`，无 `:active` 反馈，点击不增加金币）
  - 历史页的「出差」复选框被禁用
- **历史记录**：
  - 列表只包含至少完成过 1 个时间块的日期
  - 翻页浏览（`‹` / `›`）不会把日期写入历史
  - 启动时会自动清理「全无完成」的日期
- **重置**：
  - 庆祝页的「重置当日」会清空当天打卡，并从全局金币扣回当天已得金币
  - 顶栏的状态栏在历史页隐藏
- **旧数据兼容**：
  - 旧版 `time` 字段在读取时自动迁移为 `timeStart`
  - 模板变更 / 数量不符时，会按当前 weekday + 出差状态重排日程
  - 重排时**保留同名块的打勾状态**（仅按 `label` 匹配）
  - 损坏的 JSON 不会白屏，会回退到默认计划

## 数据结构

保存在浏览器 `localStorage`：

| Key | 含义 |
|---|---|
| `dayplan_YYYY-MM-DD` | 某天的完整计划（详见下方） |
| `dayplan_coinCount` | 全局累计金币（整数字符串） |
| `dayplan_date` | 当前查看的日期（刷新后恢复） |
| `dayplan_allDates` | 至少完成过 1 个时间块的日期列表（JSON array） |

某天计划：

```json
{
  "date": "2026-05-30",
  "trip": false,
  "workflow": "workday",
  "blocks": [
    {
      "id": 0,
      "timeStart": "06:40",
      "timeEnd": "07:10",
      "label": "游泳",
      "period": false,
      "done": false,
      "doneAt": null
    },
    {
      "id": 1,
      "timeStart": "白天",
      "timeEnd": null,
      "label": "按出差流程工作",
      "period": true,
      "done": false,
      "doneAt": null
    }
  ],
  "coinCount": 3
}
```

字段说明：
- `workflow`：`workday` | `saturday` | `sunday` | `trip`
- `period`：`true` 表示时段块，无具体结束时间，不显示时长
- `timeStart` / `timeEnd`：24h `HH:MM`；时段块 `timeEnd = null` 且 `timeStart` 不是 `HH:MM` 格式
- `done` / `doneAt`：完成状态与完成时间（ISO 字符串）
- `coinCount`：当日获得金币数（`dayplan_coinCount` 是全局累计）

## 时长显示规则

卡片右侧的副标题（如 `30′`、`3h`、`3h30′`）按下面规则生成：

| 时长 | 显示 |
|---|---|
| < 60 分钟 | `30′`、`50′` |
| 整小时 | `3h`、`1h` |
| 小时 + 分钟 | `3h30′`、`1h30′` |
| 时段块（`period: true`） | 不显示 |
| `timeEnd` 缺失 | 不显示 |

跨午夜按次日计算（`22:40 → 05:40` 视为 7 小时）。

| 视口 | 行为 |

| < 640px | Tab 文案短版 `今日·工作日` / `历史`；进度文案短版 `1/3 · 6/2 · 工作日`；时间轴列宽按比例缩小 |

## 无障碍

- Tab 用 `role="tablist"` / `role="tab"` / `aria-selected` 标记
- 进度区用 `aria-live="polite"`，完成块数变化时读屏器会播报
- 装饰性图标（`🪙`、`✓`、`🎉`）用 `aria-hidden="true"`
- 历史列表项 `role="button"`，支持回车 / 空格键激活
- 复选框 / 按钮均带 `aria-label`，无文字按钮有明确标签
